Issues found

1
Extra large avatar size mismatch Critical

The Figma Extra large Avatar (both Person and Entity) is 56px. The Storybook xl size resolves to 80px via uy:size-1000. This is the most impactful discrepancy — the component renders at a notably larger size than designed.

Figma — Extra large
56 × 56px
Storybook — xl
80 × 80px (uy:size-1000)
Expected Align on a single value. Either update xl to use a 56px spacing token, or update the Figma design to 80px and re-verify all layout usages.
2
Extra large square — hardcoded border-radius, no token Major

The Figma Extra large Entity uses a hardcoded 11.2px border-radius with no design token reference. The Storybook implementation uses uy:rounded-200 = 16px. The visual ratio is consistent at ~20% of the container, but the Figma value is not tokenised.

Figma — Extra large square
11.2px (hardcoded)
Storybook — xl square
uy:rounded-200 = 16px
Expected Figma should reference var(--corner-radius/radius-200, 16px). Once size is aligned (issue #1), both values should match.
3
Icon size inside Avatar doesn't match Figma Major

The Figma Icon component uses a fixed 20px icon across all sizes, varying only the outer padding. Storybook AvatarIcon scales the icon proportionally with the avatar size. This produces different icon-to-container ratios at sm and lg.

Avatar size Figma icon size Storybook icon size
sm (24px) 20px 16px Mismatch
md (32px) 20px 20px Match
lg (40px) 20px 24px Mismatch
xl ~14px 40px Mismatch
Expected Decide on one approach: fixed 20px icon (Figma) or proportional scaling (Storybook). Update the implementation or design accordingly so all sizes are consistent.
4
Decorative text color tokens — wrong tier referenced in Figma Major

For some color variants, Figma references default-tier content tokens while Storybook uses high-tier tokens (darker, higher-contrast colors). This affects Purple, Orange, and Primary text in icon/fallback avatars.

Color Figma token (default) Storybook hex (high tier)
Purple text content/decorative-purple/default = #7b62ff #291a77
Orange text content/decorative-orange/default = #f86544 #842c19
Primary text content/primary/default = #0a73eb #084385
Expected Figma should reference the high-tier tokens (content/decorative-purple/high, content/decorative-orange/high, content/primary/high) to match what is implemented for better contrast.
5
Avatar size xs (16px) in Storybook — not documented in Figma Minor

The Storybook exposes an xs (16px) size for the main Avatar component. In Figma, 16px only appears as the AvatarPair sub-component — it is not a standalone Avatar size in the component spec.

Expected Either add a corresponding xs size to the Figma design documentation, or restrict its usage in Storybook to the AvatarPair context only.
6
Placeholder blue color in Storybook — absent from Figma component Minor

The Storybook AvatarFallback placeholder variant has three colors: sunglow, peach, and blue (uy:bg-blue-l7 = #0d62c4). The Figma only shows the first two. A note in the Figma frame reads: "New decorative colour 'shades-150' will have to be added to the token library — owned by Brand team." The blue/L7 token exists in the Figma variables but the variant is not shown in the published component.

Figma placeholder variants
sunglow, peach
Storybook placeholder variants
sunglow, peach, blue
Expected Add the blue placeholder variant to the Figma component once the Brand team confirms the token, or remove it from the implementation until the design is finalised.
7
AvatarPair behavior for xl parent — not documented in Figma Minor

In Storybook, when the parent Avatar is xl, the AvatarPair automatically switches to sm (24px) size with 0px offset. Figma only shows the 16px pair at a fixed position and documents no behavior for an Extra large parent.

Expected Document the xl parent behavior in Figma: pair size = 24px, offset = 0px.
8
Fallback typography at lg and xl — not specified in Figma Minor

The Storybook AvatarFallback changes typography at larger sizes — lg uses body-strong (16px/500) and xl uses h2 (22px/600). The Figma design only documents the Small Avatar initials at 12px. No typographic specs are provided for lg or xl initials.

Size Figma Storybook
sm / md 12px, weight 500 body-small-strong — 12px / 500
lg Not specified body-strong — 16px / 500
xl Not specified h2 — 22px / 600
Expected Add typography specs for lg and xl initials fallback to the Figma component documentation.
